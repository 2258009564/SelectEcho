# 贡献指南

感谢你愿意为 SelectEcho 做贡献。

本项目优先追求三件事：稳定、可解释、可维护。请在提交前阅读以下规则。

## 先看这个项目的基本约束

1. 引擎策略保持“百度或 Google”二选一，不引入多第三方 fallback。
2. 当 Google 链路异常时，优先返回友好错误提示，而不是继续切换不稳定节点。
3. 保持中文逻辑约束：繁体中文翻译为简体中文，简体中文默认跳过。
4. 结构保真优先：尽量保留段落、列表、强调、换行，不为“过度美化”牺牲可读性。

## 如何参与

### 方式 1：提 Issue

适用于 bug 报告、需求建议、文档改进。

建议包含：

1. 浏览器与版本（Chrome/Edge + 版本号）
2. 扩展版本（manifest 里的 version）
3. 复现步骤（尽量最小化）
4. 期望结果与实际结果
5. 关键日志或截图（可选）

### 方式 2：提 Pull Request

适用于你已经实现改动。

建议流程：

1. Fork 仓库并创建分支：
   - 功能：`feat/xxx`
   - 修复：`fix/xxx`
   - 文档：`docs/xxx`
2. 在本地完成开发与自测。
3. 提交 PR，清晰描述变更内容、影响范围和测试结果。
4. 通过 review 后合并。

## 本地开发与自测

1. 在浏览器打开扩展管理页：
   - Chrome：`chrome://extensions`
   - Edge：`edge://extensions`
2. 打开“开发者模式”，加载项目目录。
3. 每次修改后点击扩展卡片“刷新”，再刷新目标网页。

最小自测清单：

1. `auto` / `baidu` / `google` 三种模式至少各测一次。
2. 简体文本应跳过，繁体文本应继续翻译为简体。
3. 输入框、textarea、contenteditable 内不应误触发翻译。
4. 富文本场景（列表、强调、换行）应保持基本结构。
5. 异常场景（超时、凭证缺失）应有清晰错误提示。

## 代码规范

1. 不做与需求无关的大规模重构。
2. 保持现有命名与内部前缀（例如 `dst-*`）稳定，避免破坏兼容性。
3. 注释要说明“为什么”和“规则边界”，避免无意义注释。
4. 新增文案请与 README、设置页说明保持一致。

## 提交信息建议

推荐使用清晰前缀：

- `feat:` 新功能
- `fix:` 修复问题
- `docs:` 文档变更
- `refactor:` 重构（无行为变化）
- `chore:` 构建/流程类改动

示例：

- `feat: add release zip workflow`
- `fix: handle google timeout message`
- `docs: split contributing guide into standalone file`

## PR 合并前检查

请确认：

1. 行为与 README 描述一致。
2. 未破坏“二选一引擎”核心策略。
3. 关键改动有自测说明。
4. 无明显语法错误或运行时错误。

## 安全问题

如果你发现安全漏洞，请不要公开提 issue。

请通过私下方式联系维护者处理，再决定公开披露时间。
